# Peerpedia 内容策略

> 2026-06-06 — 用 Peerpedia 写科普和历史，自测流程，打磨产品

## 定位

Peerpedia 在这个阶段的生态位不是"取代出版社"，而是**从另一边切入**——不抢发表市场，抢**内容消费**市场。科普/历史的读者比学术论文的读者多三个数量级。先用好内容把产品打磨到精致，再向上渗透到学术圈。

**竞争差异：**

| | 传统出版 | arXiv | Wikipedia | Peerpedia |
|---|---|---|---|---|
| 质量控制 | 编辑垄断 | 无 | 编辑战 | 社区评分+沉淀池 |
| 发表速度 | 6-18 月 | 即时 | 即时 | 即时→沉淀→出池 |
| 读者信任 | 品牌背书 | 个人判断 | 不可引用 | 评分+评审+信誉 |
| 内容类型 | 论文 | 预印本 | 百科 | 科普+论文+笔记 |
| 版本历史 | 无 | 无（只有 v1/v2） | 有 | Git 全历史 |
| 五维评分 | 无 | 无 | 无 | ✅ |

## 核心洞察

1. **GitHub 模式可以复用到学术内容** — fork/merge/commit 不是程序员专属，科普作者也需要
2. **沉淀池 = 质量滤镜** — 评分的核心不是排名，是大众筛选
3. **五维评分是差异化** — 一篇文章可以"原创很强但不好读"，或者"很好读但不够严谨"。一维评分太粗糙
4. **Thread 对话 = 学术讨论的存档** — 多人的反复交流比单次评审有价值

## 内容方向

科普和历史，风格是"大框架解释"——不讲这个定理怎么证，讲这个东西为什么是这样。

### 备选篇目

1. **《学术出版的中心化分合》** — 从 17 世纪 gentleman's club 到 Elsevier 寡头垄断
2. **《弗洛伊德为什么说性是驱动力》** — 19 世纪遗留问题，不是科学结论
3. **《分析哲学和欧陆哲学为什么分家》** — Cassirer 和 Heidegger 的达沃斯之辩
4. **《Grothendieck 的传奇》** — 不是数学，是拒绝这个世界的规则
5. **《费米液体为什么成功》** — 一个违背直觉的简化为什么能解释几乎所有金属
6. **《同行评审的诞生》** — 三个匿名评委怎么获得审判你论文的权力
7. **《SCI 怎么变成学术绞肉机》** — Eugene Garfield 1955 年的一篇 2 页短文
8. **《弗洛伊德和荣格决裂的真正原因》** — 不全是性，是认识论的战争
9. **《行为主义的恐怖统治》** — Skinner 和 Chomsky 1959 年的认知革命
10. **《为什么法国人觉得拉康不是骗子》** — 法国知识场域的结构
11. **《冯·诺依曼为什么什么都做》** — 不是聪明，是看见了别人看不见的连接
12. **《Landau 学派》** — 苏联理论物理的黄金时代为什么再也回不来了
13. **《命名为什么重要》** — 林奈如何改变了现代人的思维结构
14. **《地图、统计图和线》** — Playfair 1786 年以前世界上没有统计图
15. **《为什么失败的理论不会死》** — Lakatos 和科学退行纲领
16. **《西罗马灭亡不能怪蛮族》** — 税收崩溃才是根本

## 自测流程

1. 注册 1 个主账号，写你自己
2. 从备选篇目挑 1 篇，写完，自评，发布到沉淀池
3. 走完 `self-test-checklist.md` 所有 12 个模块
4. 每发现 1 个问题 → 修 → commit
5. 边写边记 bug

## 长期：吸引贡献者

三个月目标：产品能独立自测，有一个公网 URL，朋友点开即用。

| 阶段 | 时间 | 目标 |
|------|------|------|
| 自测 | 1-2 周 | 写完 3-5 篇文章，修完 checklist 上的 bug |
| 小范围 | 第 3 周 | 部署到 VPS，邀请 3-5 个朋友测试 |
| 公开 | 第 4-6 周 | 优化体验，发布 HN/Reddit/知乎 |
