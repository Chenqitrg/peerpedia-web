# 线下设计需求

- 用户必须登录才能使用这个软件
- 但是登录之后，导航栏的右边，整个界面的右上方有一个网络图标，它标记了当前网络状态
- 单机时，除了schools不能看，其余用户的关注与被关注不能看，沉淀池不能看之外，其余服务都能用
- 总之，基本原理是，用户在线下只能看到自己在意的，不需要协作，不反映实时状态的东西

## 首页

- 首页的推荐文章（feed）只列出自己看过的
- 收藏夹里全显示，收藏意味着默认缓存
- 搜索：在自己的所有数据库中搜索，包含所有缓存过的文章
- 线上搜索：除了所有自己的数据库，还包含了我关注的人的文章
- 沉淀池：看不到任何消息，显示无网络，进不去
- schools：看不到任何人，显示无网络，进不去
- 个人主页：完全与线上相同，和自己所有相关的东西都是默认保存的例如自己的文章，我的关注与关注我的
- 新建文章：可以访问，但有一点功能受限

## 文章编辑页

- 除了发布到沉淀池之外，所有功能正常使用
- 发布到沉淀池变成灰色，直接阻挡用户点进去
- 保存文章的话保存在本地，可以在个人主页中找到
- 编译，下载都直接调用本地工具

## 个人主页

- 与线上完全相同
- 可以看到我关注的与关注我的，但是进一步跳转（朋友的朋友）受限
- 我的评分继承上次联网时的分数，所有文章同理
  
## 文章页面

- 文章页面大部分继承线上功能
- 不能看的：如果作者未缓存，不能看作者的个人主页
- 或许不能fork（还需要确认，因为fork需要沟通），这里的fork应该是集成了pull与fork？
- 源码，预览什么的都能做，默认文章本本地缓存了
- 评价历史也能看到，但是看不到更新
- 评论功能受限，需要联网才行
- 历史页正常，就是记录了这个仓库截止到缓存那一刻的历史
