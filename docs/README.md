# Docs

> PeerPedia 项目文档。按类型组织，不是按时间。

## 目录

| 目录 | 内容 | 什么时候看 |
|------|------|-----------|
| [vision.md](vision.md) | 项目为什么存在 | 第一次来、忘了初衷 |
| [architecture/](architecture/) | 系统怎么设计的——DESIGN 文档 + 架构手册 8 章 | 需要理解全局设计 |
| [decisions/](decisions/) | 架构决策记录（ADR）——重大选择及其原因 | 想改架构、想知道"当初为什么这么搞" |
| [learn/](learn/) | 个人理解——用自己的语言重述每个模块 | 接手一个模块前、忘了一个模块怎么工作 |
| [plans/](plans/) | 实施计划——每个 feature/bugfix 的分步方案 | 开始写代码前 |
| [specs/](specs/) | 功能规格——每个功能的设计规格 | 实现新功能、理解旧功能的设计意图 |
| [reviews/](reviews/) | 代码评审 + 调试复盘 | 想了解代码质量现状 |
| [archive/](archive/) | 过期/临时文档 | 基本不看 |
